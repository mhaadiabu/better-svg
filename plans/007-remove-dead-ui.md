## Why this matters

`packages/ui` is Better-T-Stack scaffolding that was never wired into the app.
It has a `components.json` (shadcn config), a full set of shadcn components
(button, card, checkbox, dropdown-menu, input, label, skeleton), `hooks/`,
`lib/`, `styles/`, and 8 dependencies (`@base-ui/react`, `class-variance-authority`,
`clsx`, `lucide-react`, `next-themes`, `shadcn`, `sonner`, `tailwind-merge`,
`tw-animate-css`) — none of which are used by the published `@mhaadi/svg`
package or the `apps/web` Astro site. The only `@better-svg/ui` imports in the
repo are **intra-package** (its own components importing its own `lib/utils`).
Removing it shrinks the install graph, removes a stale shadcn/tailwind config
that could mislead contributors, and makes the workspace honest about what
exists. It's low-risk because nothing imports it from outside.

## Steps

### Step 1: Confirm no external dependents

Run a grep to confirm nothing outside `packages/ui` imports it:
```
grep -rn "@better-svg/ui" --include="*.ts" --include="*.tsx" --include="*.astro" --include="*.mjs" . --exclude-dir=node_modules --exclude-dir=dist --exclude-dir=.turbo
```
Every match should be inside `packages/ui/src/`. If any match is outside
`packages/ui`, **STOP** — the package is not dead; report the dependent file.

**Verify**: the grep output contains only `packages/ui/src/...` paths.

### Step 2: Delete the package directory

```
git rm -r packages/ui
```
This stages the deletion. Do not use plain `rm` — `git rm` keeps the index in
sync and makes the deletion explicit in the diff.

**Verify**: `test ! -d packages/ui` → exit 0. `git status` shows `packages/ui` deletions.

### Step 3: Reinstall to update the lockfile

```
pnpm install
```
pnpm will prune `@better-svg/ui` and its exclusive deps from `node_modules`
and update `pnpm-lock.yaml`. Shared deps (react, etc., used by `@mhaadi/svg`)
stay.

**Verify**: `pnpm install` → exit 0. `git status` shows `pnpm-lock.yaml` modified. `test ! -d packages/ui/node_modules` → exit 0 (pruned).

### Step 4: Full verification

**Verify**:
- `pnpm check-types` → exit 0 (turbo runs one fewer package; `@mhaadi/svg` and `web` still pass).
- `pnpm build` → exit 0.
- `pnpm lint` → exit 0.
- `pnpm test` → all pass (no tests lived in `packages/ui`).
- `grep -rn "@better-svg/ui" . --exclude-dir=node_modules --exclude-dir=dist --exclude-dir=.turbo --exclude-dir=.git` → **no matches** (the package and all its self-references are gone).
